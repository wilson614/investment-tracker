# Implementation Plan: Unified Broker Statement Import

**Branch**: `012-import-broker-statement` | **Date**: 2026-02-15 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/012-import-broker-statement/spec.md`

## Summary

Extend the existing stock CSV import entry to support both legacy CSV and Taiwan broker statement CSV in one flow, with format detection + user override, import preview, on-demand security mapping resolution, and explicit insufficient-balance decisions (Margin / Top-up) aligned with manual transaction creation. Reuse current CSV preview UI and existing single-transaction business rules while adding a stock-import batch contract that returns deterministic row-level diagnostics and execution summary.

## Technical Context

**Language/Version**: C# .NET 8 (backend), TypeScript 5.x + React 18 (frontend)
**Primary Dependencies**: ASP.NET Core 8, Entity Framework Core, FluentValidation, React 18, TanStack Query, Vitest
**Storage**: PostgreSQL
**Testing**: xUnit (backend), Vitest + React Testing Library (frontend)
**Target Platform**: Linux containerized web deployment (self-hosted)
**Project Type**: Web application (frontend + backend)
**Performance Goals**: Import preview for a 500-row CSV completes within 3 seconds on local environment; execution returns full per-row result in a single response
**Constraints**:
- One import entry must support legacy CSV and broker statement CSV (FR-001, FR-012)
- Buy rows with insufficient balance must require explicit `BalanceAction` choice before execution (FR-008~FR-011)
- Security resolution uses local mapping first; unresolved rows trigger on-demand TWSE sync, then manual ticker input if still unresolved (FR-006~FR-006d)
- Synchronization failure must not block already resolvable rows (FR-006c)
- Financial behavior must remain consistent with `CreateStockTransactionUseCase` balance logic
- No introduction of mandatory always-online dependency for core import path
**Scale/Scope**:
- New stock import batch workflow and DTO contracts
- New TW security mapping persistence + on-demand synchronization flow
- Frontend import interaction changes across preview, unresolved-row handling, and insufficient-balance decision UX
- Regression coverage for legacy CSV path and manual-creation parity rules
